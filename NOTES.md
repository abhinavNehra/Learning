# Teaching Notes

## User Profile
- Senior Full-Stack Developer, 11 years, React / Node.js / Python
- Completed: 6-module Prompt Engineering curriculum (tokens, temperature, CoT, system prompts, RAG basics, evals)
- Working env: macOS, VS Code, .venv set up, Anthropic SDK installed
- Goal: Full AI expertise (use → understand → modify → deploy)

## Preferences
- Hands-on code first, theory second — never open with abstract concepts
- Lessons that feel too "shallow" or "just API calls" will disengage him
- He wants to understand HOW things work, but not before he can USE them
- Serious time commitment: 4–8 hrs/week

## Teaching Approach
- Start every lesson with a working code demo — the theory explains WHY the code does what it does
- Use his React/Node.js mental models as analogies (e.g. React state tree → transformer attention)
- Quiz questions should require recall, not recognition — avoid "which of these is true" formats
- Each lesson should produce something he could show to a colleague

## Phase Progression
1. **AI Builder** (now) — Agents, RAG, production patterns, multi-modal
2. **Model Whisperer** — Transformers, attention, architecture internals
3. **Model Modifier** — Fine-tuning, LoRA, RLHF
4. **Production AI** — Model serving, MLOps, deployment
5. **Expert** — Research literacy, multi-agent systems, staying current
